# 开发文档整理说明

本文档说明开发文档的组织结构和整理原则。

## 📁 文档组织结构

所有开发相关文档已整理到 `docs/dev/` 目录下，包括：

### 核心流程文档
- **流程与LLM交互索引.md** - 最新的完整流程文档（推荐阅读）
- **每轮游戏流程说明.md** - 游戏循环详细说明
- **流程与LLM交互索引_整理版.md** - 历史整理版本（仅供参考）
- **流程说明.md** - 简化的流程说明（历史版本，仅供参考）

### 数据格式文档
- **传递给LLM的信息说明.md** - LLM输入输出格式详细说明
- **LLM返回JSON格式详细说明.md** - LLM返回的JSON格式说明
- **导演决策返回格式说明.md** - 导演评估返回格式说明

### 开发计划文档
- **剧本系统开发计划.md** - 剧本系统的开发计划
- **剧本系统更新确认.md** - 剧本系统更新确认文档

### 问题分析文档
- **交互索引文档问题分析.md** - 交互索引文档的问题分析
- **文档问题总结.md** - 文档问题的总结

## 📝 整理原则

1. **保留历史版本**：历史版本的文档保留作为参考，但标记为"仅供参考"
2. **明确当前版本**：最新版本的文档明确标记为"推荐阅读"
3. **分类组织**：按文档类型（流程、格式、计划、问题）分类
4. **索引导航**：通过 README.md 提供清晰的导航

## 🔄 文档维护

- **更新时机**：代码有重大变更时，应同步更新相关文档
- **版本标记**：重要变更应在文档中标记版本或日期
- **问题记录**：发现的问题应记录在问题分析文档中

## ⚠️ 注意事项

1. **不要删除历史文档**：历史文档保留作为参考，即使已过时
2. **保持文档同步**：文档应与代码实现保持一致
3. **及时更新索引**：新增文档时应更新 README.md
